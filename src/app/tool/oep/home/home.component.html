OpenEfficiency Platform (OEP) is a U.S. Department of Energy-funded initiative to help commercial whole building energy efficiency programs be more efficient and gain more value from data. It involved pilots that focused on exploring use cases for existing Federal tools and industry standards and their integration to support programs. The graphic below shows how OpenEfficiency Platform tools integrate with each other and with other building energy data tools, such as SEED, Asset Score, and ENERGY STAR Portfolio Manager:

The OEP GitHub repository houses open source resources developed as part of this initiative; It includes folders for documentation of a full Benchmark use case, the OEI Salesforce Manged Package, and sample code for connecting to Energy Design Assistance Program Tracker (EDAPT), ENERGY STAR Portfolio Manager (ESPM), and Green Button Connect (GBC) OAuth 2.0:

  • SEED Benchmark: Dockered MuleSoft API for SEED to Salesforce OEP Managed Package benchmark use case. (This solution includes an Implementation Guide and a User Guide)
  • Salesforce Package: Salesforce OEP Managed Package that supports efficiency program and benchmark use cases, is BEDES compliant, and includes fields mapped from ESPM, EDAPT, and Asset Score
  • GBC OAuth 2.0: MuleSoft API OAuth 2.0 example for Green Button Connect (GBC)
  • ESPM: MuleSoft API example connection between ESPM and Salesforce
  • EDAPT: MuleSoft API example connection between EDAPT and Salesforce

In addition to the open source tools, OEP has published case studies on how the Platform was able to help the cities of San Francisco and Berkeley. Performance Systems Development (PSD) is available to provide support for the configuration and hosting of these tools on a fee-for-service basis.
